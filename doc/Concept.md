# AsciiArtify – Kubernetes Local Dev Concept

## 1. Вступ

Для локальної розробки Kubernetes було розглянуто три інструменти:

- **minikube** — локальний кластер Kubernetes (VM або контейнер)
- **kind** — Kubernetes in Docker (кластер у контейнерах)
- **k3d** — lightweight Kubernetes (k3s) у Docker

Мета: вибрати оптимальний інструмент для PoC стартапу.

---

## 2. Характеристики

| Характеристика | minikube | kind | k3d |
|----------------|----------|------|-----|
| Тип | VM/Container | Docker | Docker (k3s) |
| Швидкість | повільніше | швидко | дуже швидко |
| RAM usage | високий | середній | низький |
| Multi-node | так | так | так |
| CI/CD | обмежено | добре | дуже добре |
| ARM support | так | так | так |
| Моніторинг | addons | вручну | вручну |
| Простота | середня | проста | дуже проста |

---

## 3. Переваги та недоліки

### 🔹 minikube

✅ Плюси:
- максимально наближений до реального Kubernetes
- багато addons

❌ Мінуси:
- повільний
- потребує більше ресурсів

---

### 🔹 kind

✅ Плюси:
- швидкий
- ідеальний для CI/CD
- простий

❌ Мінуси:
- обмежені можливості networking
- немає built-in addons

---

### 🔹 k3d

✅ Плюси:
- дуже швидкий
- легкий
- простий у використанні
- ідеальний для локальної розробки

❌ Мінуси:
- не 100% upstream Kubernetes (k3s)

---

## 4. Docker vs Podman

Docker має ліцензійні обмеження для бізнесу.

Альтернатива:
- **Podman**
- сумісний з Docker CLI
- rootless режим

Висновок:
✔ для стартапу краще використовувати Podman або Docker Personal
